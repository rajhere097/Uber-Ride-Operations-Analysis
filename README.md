Uber Ride Operations Analysis
Cancellation, Revenue & Demand Insights
Overview

This project analyzes 150K Uber ride bookings to identify cancellation drivers, demand patterns, and revenue loss using SQL, Python, statistical testing, machine learning, and Power BI dashboards.

The goal is to uncover operational inefficiencies and provide data-driven recommendations to reduce ride cancellations and improve revenue.

Tools & Technologies

Python (Pandas, Scikit-learn, Matplotlib)

SQL (MySQL)

Power BI

Statistical Hypothesis Testing

Logistic Regression

Key Insights

25% of rides were cancelled

Estimated ₹18.9M revenue loss due to cancellations

Certain locations and time slots show higher cancellation rates

Ride distance and operational factors significantly influence cancellations

Machine Learning Model

A Logistic Regression model was built to predict ride cancellations.

Model performance:

ROC-AUC Score: 0.95

indicating strong ability to distinguish cancelled vs completed rides.
